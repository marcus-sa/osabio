The most efficient architecture is to use Better Auth as the Identity Provider (IdP) and your custom server as the Authorization Server (AS).

The Strategy: "Better Auth" as the Foundation

You don't throw away Better Auth. You use it to handle the "Human" side of the equation (Login, MFA, Session Management), while your custom server handles the "Agent" side (Token issuance, RAR scopes, DPoP binding).

Better Auth's Role: It authenticates the Human Owner. When an agent needs a token, the user logs in via Better Auth. Better Auth then passes a userId or session to your custom server.

Your Server's Role: It takes that authenticated session and mints a DPoP-bound JWT with a RAR authorization_details payload.
