# Jobs-to-be-Done: Coding Agent Orchestrator

## Job 1: Assign Task to Coding Agent

**Job Story:** When I have a well-defined engineering task in my workspace, I want to assign it to a coding agent with one click, so I can focus on higher-value work while routine coding gets done.

### Job Dimensions

| Dimension | Description |
|-----------|-------------|
| **Functional** | Delegate an engineering task to an AI coding agent that has full project context |
| **Emotional** | Feel confident that the task will be handled competently without babysitting |
| **Social** | Be seen as someone who leverages AI effectively to multiply team output |

---

## Job 2: Monitor Agent Progress

**Job Story:** When a coding agent is working on my task, I want to see what it's doing in real-time, so I can catch problems early and feel confident the work is on track.

### Job Dimensions

| Dimension | Description |
|-----------|-------------|
| **Functional** | See agent actions (file changes, tool calls, reasoning) as they happen |
| **Emotional** | Feel in control, not anxious about a black box |
| **Social** | Be able to explain to teammates what the agent is doing and why |

---

## Job 3: Review and Accept Agent Output

**Job Story:** When the coding agent finishes a task, I want to review what it built with full context on its decisions, so I can merge good work quickly or course-correct efficiently.

### Job Dimensions

| Dimension | Description |
|-----------|-------------|
| **Functional** | See the diff, understand agent reasoning, approve or reject the output |
| **Emotional** | Trust the review process enough to merge with confidence |
| **Social** | Maintain code quality standards the team expects |
