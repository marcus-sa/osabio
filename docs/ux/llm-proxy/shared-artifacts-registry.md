# Shared Artifacts Registry: LLM Proxy

**Epic**: llm-proxy
**Last Updated**: 2026-03-15

---

## Registry

### proxy_url
- **Source of truth**: Workspace settings (persisted in workspace record)
- **Consumers**: brain init (generates env vars), agent env vars (ANTHROPIC_BASE_URL), dashboard status indicator
- **Owner**: Workspace settings module
- **Integration risk**: HIGH -- mismatch breaks all agent routing
- **Validation**: brain init must read from workspace settings; dashboard must display same URL; health check endpoint must respond at this URL

### workspace_id
- **Source of truth**: Workspace record in SurrealDB
- **Consumers**: X-Brain-Workspace header, authentication step, trace edges (scoped_to), spend counters, dashboard filtering, audit queries
- **Owner**: Workspace module
- **Integration risk**: HIGH -- wrong workspace means misattributed costs and wrong policy evaluation
- **Validation**: Header value must resolve to an active workspace; trace edges must reference same workspace; spend counters must be scoped to same workspace

### session_id
- **Source of truth**: Claude Code metadata.user_id field (format: user_{hash}_account_{uuid}_session_{uuid})
- **Consumers**: Trace edges (invoked), session cost aggregation, anomaly detection, audit provenance chain
- **Owner**: Proxy authentication step (parser)
- **Integration risk**: HIGH -- missing session = unattributed calls, broken provenance
- **Validation**: Regex parse must extract valid UUID; trace edges must reference parsed session ID; aggregation queries must group by same ID

### task_id
- **Source of truth**: X-Brain-Task header (set by brain start or manually)
- **Consumers**: Trace edges (attributed_to), task cost rollup, project-level cost aggregation
- **Owner**: brain CLI (brain start command)
- **Integration risk**: MEDIUM -- optional, but when present must match a valid task record
- **Validation**: If present, must resolve to an existing task in the workspace; trace edges must reference validated task ID

### model_id
- **Source of truth**: Request body .model field
- **Consumers**: Policy check (model access), cost calculation (pricing lookup), trace record, analytics
- **Owner**: Client (Claude Code / agent)
- **Integration risk**: HIGH -- wrong model ID means wrong cost and wrong policy evaluation
- **Validation**: Model ID must exist in pricing table; policy check must use same model ID as forwarded request; trace must record actual model from response (may differ from request)

### call_cost
- **Source of truth**: Computed by proxy from usage tokens + pricing table
- **Consumers**: Spend counters, monitoring dashboard, audit data, anomaly detection
- **Owner**: Proxy cost calculation module
- **Integration risk**: HIGH -- inaccurate cost destroys trust in the entire system
- **Validation**: Cost = (input_tokens * input_rate + output_tokens * output_rate + cache_create * cache_write_rate + cache_read * cache_read_rate) / 1_000_000; spend counter increments must equal individual trace costs

### pricing_table
- **Source of truth**: Local configuration file (updated when Anthropic changes pricing)
- **Consumers**: Cost calculation module
- **Owner**: System configuration
- **Integration risk**: MEDIUM -- stale prices cause cost drift (but are self-correcting on update)
- **Validation**: Pricing table must have entries for all models the proxy forwards to; spot-check computed costs against Anthropic's usage dashboard monthly

### policy_graph
- **Source of truth**: Brain policy engine (policy table in SurrealDB)
- **Consumers**: Authorization step, audit provenance chain
- **Owner**: Policy module
- **Integration risk**: HIGH -- stale or missing policies mean wrong enforcement decisions
- **Validation**: Policy evaluated for each request must have status "active"; policy reference in error responses must be a real policy ID; audit trail must link to the evaluated policy

### trace_id
- **Source of truth**: Generated by proxy (UUID per LLM call)
- **Consumers**: Provenance chain, audit queries, session aggregation, dashboard drill-down
- **Owner**: Proxy trace capture module
- **Integration risk**: LOW -- generated fresh per call, no cross-system dependency
- **Validation**: Unique per call; all edges reference same trace ID; audit queries return traces by ID

### request_id
- **Source of truth**: Anthropic response header (request-id)
- **Consumers**: Trace record (for cross-referencing with Anthropic's billing), debugging
- **Owner**: Anthropic API
- **Integration risk**: LOW -- informational, used for debugging only
- **Validation**: Stored in trace record; matches Anthropic's request log when investigated

### spend_counters
- **Source of truth**: Aggregated from llm_trace nodes (or maintained as running counters)
- **Consumers**: Budget enforcement (authorization step), monitoring dashboard, alert thresholds
- **Owner**: Proxy spend tracking module
- **Integration risk**: HIGH -- spend counters out of sync with traces = wrong budget enforcement
- **Validation**: Counter values must equal SUM(cost_usd) from corresponding llm_trace nodes; reconciliation check runs periodically

---

## Consistency Rules

1. **proxy_url** configured in workspace settings MUST equal what brain init writes to ANTHROPIC_BASE_URL
2. **workspace_id** in X-Brain-Workspace header MUST match the workspace owning the proxy instance
3. **session_id** parsed from metadata.user_id MUST be the same value used in trace edges and session aggregation
4. **model_id** from request body MUST be the same value used for policy check, cost calculation, and trace record
5. **call_cost** in each llm_trace MUST equal the cost computed from that trace's token counts and the pricing table at time of computation
6. **spend_counters** MUST equal the sum of call_cost across all traces in the corresponding scope (workspace/project/task)
7. **pricing_table** MUST have an entry for every model_id the proxy encounters; unknown models trigger a warning observation
