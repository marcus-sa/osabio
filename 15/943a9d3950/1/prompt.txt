N+1 queries inside loadObservationLinkedEntities

After fetching all linked entity IDs in a single query, the function enters a for loop and issues a separate SELECT … FROM $record DB call for each row. With N linked entities this becomes N+1 round-trips.

A single batched SELECT title, summary, text, description, meta::tb(id) AS table_name FROM $records call (where $records is the array of RecordId values) would fetch all entity details in one query. Example:

const targets = rows.map((r) => r.out);
if (targets.length === 0) return [];

const [details] = await surreal.query<[Array<{
  id: RecordId;
  title?: string; summary?: string; text?: string; description?: string;
}>]>(
  `SELECT id, title, summary, text, description FROM $records;`,
  { records: targets },
);
Under heavy peer-review load (many linked entities) the current approach will serialize round-trips and significantly slow the observer pipeline.
