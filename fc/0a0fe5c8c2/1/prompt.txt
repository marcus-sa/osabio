User comment: "**`DialogHeader` rendered outside the portal popup**

`DialogHeader` (containing `DialogTitle` and `DialogDescription`) is placed outside `DialogContent`. Since `DialogContent` renders via a `DialogPortal`, the header will be mounted in the calling component's DOM location, not inside the dialog popup element. This means the accessible name/description won't be properly associated with the popup via `aria-labelledby`/`aria-describedby`, breaking screen-reader semantics even though the content is `sr-only`.

Move the header inside `DialogContent`:

```suggestion
  return (
    <Dialog {...props}>
      <DialogContent
        className={cn(
          "top-1/3 translate-y-0 overflow-hidden rounded-xl! p-0",
          className
        )}
        showCloseButton={showCloseButton}
      >
        <DialogHeader className="sr-only">
          <DialogTitle>{title}</DialogTitle>
          <DialogDescription>{description}</DialogDescription>
        </DialogHeader>
        {children}
      </DialogContent>
    </Dialog>
  )
```"
