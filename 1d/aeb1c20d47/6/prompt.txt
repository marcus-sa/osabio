"questionable" peer review verdict silently mapped to "info" severity

The severity ternary only distinguishes "unsupported" from everything else:

const severity = llmVerdict.verdict === "unsupported" ? "warning" as const
  : "info" as const;
This means "sound" → "info" and "questionable" → "info" — the same severity. But the LLM prompt definition for "questionable" is "partial support, some claims go beyond the evidence", which is meaningfully different from "sound" and clearly warrants a "warning" to signal that human review is needed. As written, a peer-reviewed observation where the LLM found only partial support is indistinguishable from a fully validated one at the severity level.

Suggested change
          verified: llmVerdict.verdict === "sound",
          text: llmVerdict.reasoning,
          source: "llm",
        const severity = llmVerdict.verdict === "unsupported" ? "warning" as const
          : llmVerdict.verdict === "questionable" ? "warning" as const
          : "info" as const;

Pattern synthesis receives anomalies with mismatched entity refs after dedup filtering

The anomalies array fed to synthesizePatterns is built unconditionally from contradictions, staleBlocked, and driftTasks using the raw entity IDs. But for contradictions the entity ref format is decision.id.id / task.id.id (used directly as entityId), while in anomalyCandidates for the stale/drift path the ref is the full task:${id} string.

In synthesizePatterns the format used to construct the prompt line is:

`- [${a.entityTable}:${a.entityId}] (${a.type}) ${a.text}`
For the contradiction path entityTable = "decision" and entityId = decision.id.id as string — so the prompt shows [decision:<raw-id>] without the decision: prefix that the LLM was instructed to use in detectContradictions. The LLM then returns contributing_entities in table:id format, but the contributing entities built for contradictions only have the raw ID, not the table:id string, making parseEntityRef unable to resolve them in the pattern result below.

The mismatch between how entity refs are formatted for contradictions vs anomaly candidates means pattern observations for contradictions will silently have relatedRecords: [] (empty), breaking the graph linkage that pattern synthesis is supposed to provide.
