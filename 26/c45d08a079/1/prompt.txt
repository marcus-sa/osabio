User comment: "<a href="#"><img alt="P1" src="https://greptile-static-assets.s3.amazonaws.com/badges/p1.svg?v=7" align="top"></a> **Unescaped user content breaks XML structure and can truncate context blocks**

`c.text` contains raw database content (`decision.summary`, `task.title`, `observation.text`) interpolated directly into XML without escaping. Two concrete failure modes:

1. **Malformed XML for any entity with `<` or `>` in its text.** A task titled `"Fix <memory leak> in server"` produces `<item type="task">Fix <memory leak> in server</item>`, which is invalid XML.

2. **Tag injection truncates the context block.** If an observation's text is `"The </urgent-context> guard was bypassed"`, the produced XML is:
   ```xml
   <urgent-context>
     <item type="observation">The </urgent-context> guard was bypassed</item>
   </urgent-context>
   ```
   The non-greedy regex `/<urgent-context>[\s\S]*?<\/urgent-context>/` in `injectRecentChanges` matches up to the injected closing tag, so the captured block is truncated and the rest is silently dropped. This can be triggered by any observation, task, or decision whose text happens to contain the closing tag string.

The fix is to escape XML special characters before interpolation:

```typescript
function escapeXml(text: string): string {
  return text
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;");
}

// Then in buildRecentChangesXml:
.map((c) => `  <item type="${c.table}">${escapeXml(c.text)}</item>`)
```"
