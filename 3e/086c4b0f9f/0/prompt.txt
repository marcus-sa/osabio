When the PM agent creates a new project and then immediately creates features scoped to that project (e.g., create_work_item({kind: "feature", project: "Orders"})), the has_feature edge linking the feature to the project sometimes fails to create. The error is silently swallowed by the catch block at app/src/server/chat/tools/create-work-item.ts:178. This means features exist as records but have no has_feature edge to their parent project.

The workspace field fix (this branch) makes features discoverable without the edge, but the has_feature edge is still needed for project-scoped queries (e.g., “list features in project Orders”). The edge creation failure needs to be investigated and fixed — likely a race condition in resolveWorkspaceProjectRecord not finding a just-created project, or a Surreal write consistency issue. Start by adding error logging in the catch block to surface what’s actually throwing."
