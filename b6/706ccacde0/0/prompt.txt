User comment: "**Silent skip when `SELF_HOSTED=true` but credentials missing**

`parseSeedConfig` silently returns `undefined` when `SELF_HOSTED=true` but `ADMIN_EMAIL` or `ADMIN_PASSWORD` is absent. This directly contradicts the fail-fast requirement: running `bun migrate` will exit 0 with no admin user created and no error output, while `bun start` will later fail loudly with "ADMIN_EMAIL is required when SELF_HOSTED=true". The operator gets a false green during migration and a confusing failure at startup.

`loadServerConfig()` in `config.ts` does the right thing here — it calls `requireSelfHostedEnv()` which throws immediately. The migration runner should mirror that behaviour:

```suggestion
  if (!email) throw new Error("ADMIN_EMAIL is required when SELF_HOSTED=true");
  if (!password) throw new Error("ADMIN_PASSWORD is required when SELF_HOSTED=true");
```

The corresponding unit tests in `tests/unit/runtime/admin-seed.test.ts` that assert `undefined` is returned in these cases should be updated to expect a thrown error."
